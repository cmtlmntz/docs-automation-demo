# How to Create and Format a Markdown Document

This procedure walks you through initializing a plain text Markdown (`.md`) file and applying standard markup syntax to structure your documentation.

## 1. Objective
To create a cleanly formatted, scannable document using standard Markdown tags without relying on heavy word processing software.

## 2. Prerequisites
* A GitHub account or a local plain text editor (like Notepad, VS Code, or TextEdit).
* A basic understanding of your project's repository structure.

## 3. Step-by-Step Procedure

1. **Initialize the File**
   Create a new file in your workspace or GitHub repository and name it with the `.md` extension (for example: `procedure.md`).

2. **Add the Main Document Title**
   Start the very first line of the file with a single hashtag followed by a space, then type your title. This establishes the structural Heading 1.

3. **Break the Content into Sections**
   Use double or triple hashtags to create sub-headings and subsections. This automatically builds a visual hierarchy for the reader.

4. **Insert Your Content and Lists**
   Write your instructional prose. Use numbers followed by periods for sequential steps, or dashes/asterisks to create bulleted reference blocks.

5. **Save and Render**
   Commit your changes to GitHub or save your file. The platform will automatically parse the symbols and display a styled document.

---

## 4. Markdown Syntax Quick Reference

Use these standard syntax tags to format your procedural documents:

### Headings (Section Titles)
* `# Main Title (Heading 1)` — Reserved for the absolute top title of the document.
* `## Section Title (Heading 2)` — Used for major milestones (e.g., Objectives, Steps).
* `### Subsection Title (Heading 3)` — Used for deeply nested sub-steps or specific notes.

### Lists
* **Numbered List Tag:** Type `1. `, `2. `, `3. ` at the beginning of consecutive lines to create an ordered, chronological sequence.
* **Bulleted List Tag:** Type `* ` or `- ` followed by a space to create unordered, bulleted bullet lists for tool requirements or prerequisites.

### Emphasis and Styling
* **Bold Text:** Wrap your text in double asterisks (`**your text**`) to emphasize critical actions or interface buttons.
* **Italic Text:** Wrap your text in single asterisks (`*your text*`) or single underscores (`_your text_`) for general emphasis.
* **Code Blocks:** Wrap text in backticks (`` `text` ``) to call out exact filenames, variables, or terminal commands inline.
